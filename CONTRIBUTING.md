# Contributing

This is an internal wheel-generation and audit repository.

## Change Intake

- All substantive work is tracked in `pollockjj/mydevelopment`
- Changes are expected to land through issue-scoped branches
- Trust, provenance, CI, and release changes require evidence artifacts, not summary claims

## What Belongs Here

- wheel build logic
- package specs
- audit and provenance workflows
- repository governance and validation controls

## What Does Not Belong Here

- casual feature drive-bys
- unrelated packaging experiments
- undocumented workflow or artifact changes

## Pull Request Expectations

- keep scope narrow
- include exact evidence for behavior and artifact changes
- do not weaken trust checks to make CI pass
- prefer immutable dependency references over floating ones

## Security

Use `SECURITY.md` for private reporting of security issues.
